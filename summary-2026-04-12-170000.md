## Key AI Topics Discussed

### Ralph Loops & AFK Agents
- **Sandcastle released** — Matt open-sourced [Sandcastle](https://github.com/mattpocock/sandcastle), his tool to orchestrate sandboxed coding agents. Replaces Docker Desktop frustrations with swappable sandboxes and a TypeScript-based runner.
- **Docker `sbx` vs `docker sandbox`** — New Docker MicroVMs (`sbx`) are lighter than `docker sandbox run`, come with a TUI for managing VMs, and don't require Docker Desktop. Just change one line in `afk.sh`.
- **Ralph loop debugging** — Common issue: GitHub auth not persisting inside sandbox. Fix: mount `~/.claude` into the container or re-auth inside sandbox. Also, scripts need to pass GitHub issues (not PRD/plan files) to Claude in newer versions.
- **Ralph exit issue** — If Ralph doesn't exit the loop, add to the prompt: `After committing, review the plan and check if there are remaining tasks. If the task you just completed was the last one, output <promise>NO MORE TASKS</promise>.`
- **Enhanced AFK script** — [Community-improved AFK script](https://gist.github.com/DarvilCZE/690b3b7df4657dae0fc2606d1b816589) with better visibility into what Ralph is doing under the hood.
- **Ralph vs Copilot CLI Autopilot** — Discussion comparing Ralph loops with Copilot CLI Autopilot + Fleet mode. Matt's take: "Ralph is just running a coding agent in a loop, not really a technique."

### The Smart Zone & Context Management
- **1M vs 200K context** — Matt still recommends staying in the 80-100K token range (the "smart zone") even with the new 1M context window. You can set `autoCompactWindow: 200000` in settings to cap it.
- **Context paranoia is real** — Course emphasizes plan, execute, then `/compact` as a ritual. Fresh context per Ralph iteration is the key benefit.
- **Status line** — Members building context monitors. [ccusage.com](https://ccusage.com/) and [ccunpacked.dev](https://ccunpacked.dev/) for tracking Claude Code usage.

### Skills & Workflow Patterns
- **Grill-me to PRD flow** — Matt runs `/grill-me` in a session, then says `/write-a-prd` in the same conversation. Planning to rename to `/to-prd` and `/to-issues` for more flexibility.
- **Addy Osmani's agent skills** — [github.com/addyosmani/agent-skills](https://github.com/addyosmani/agent-skills) — Production-grade engineering skills shared by Google's Addy Osmani. Very similar to Matt's setup.
- **Vercel skills repo** — [github.com/vercel-labs/skills](https://github.com/vercel-labs/skills) for sharing skills across enterprise.
- **PR review skill** — `/pr-review-comments` checks unresolved PR comments, validates and fixes them in one go.
- **Workflow pattern** — One member's proven flow: 1) `/grill-me` on unrefined items, 2) create feature PRD in `todo/` dir, 3) Ralph works on `todo/` items, 4) completed items move to `done/`.

### Model Selection & Cost Optimization
- **Opus for planning, Sonnet for execution** — Discussion on switching models based on task. Matt says "interesting idea, pretty simple and cheap to try."
- **RTK for token savings** — [rtk](https://github.com/rtk-ai/rtk) CLI proxy reduces LLM token consumption by 60-90%. One user saved 125M tokens (92% savings) across 6,176 commands.
- **Claude March 2026 double rates** — Promotion for extra usage until end of March.

### Security & Sandboxing
- **Nvidia's agentic security guide** — [Practical Security Guidance for Sandboxing Agentic Workflows](https://developer.nvidia.com/blog/practical-security-guidance-for-sandboxing-agentic-workflows-and-managing-execution-risk/)
- **Threat model for local agents** — Active debate on whether AFK is worth the security risk. Geoffrey Huntley's advice: "assume when not if for getting pwned" — use ephemeral cloud VMs.
- **macOS sandbox** — Built-in sandboxing works well without Docker. Settings in `.claude/settings.json`.
- **Node modules conflict** — Sandbox uses different (linux) node_modules vs host (darwin). Workaround: use git worktrees.

### Testing & Quality at Scale
- **Trellis engineering team** — Jay shared their approach: 50K unit tests, 9K integration tests, fully automated CI/CD, DORA metrics tracked. Anytime AI does something wrong, they add an automated guardrail (usually lint rules).
- **DORA metrics with AI** — Trellis tracking deployment frequency, lead time, and change failure rate since fully adopting agents.
- **Small PRs for AFK** — One workplace auto-approves PRs with up to 2 files and 20 lines. Discussion on how to make Ralph produce small atomic PRs.

### Prompt Engineering & Patterns
- **Augmented Coding Patterns** — [lexler.github.io/augmented-coding-patterns](https://lexler.github.io/augmented-coding-patterns/) with context markers, canary warnings, and low-level prompt patterns. Companion [video talk](https://www.youtube.com/watch?v=_LSK2bVf0Lc).
- **Prompt X-Ray** — [Tool to detect hidden characters](https://prompt-xray-tool.vercel.app/) in text before pasting into agents (prompt injection defense).
- **Caveman mode** — [github.com/JuliusBrussee/caveman](https://github.com/JuliusBrussee/caveman) — "Why use many token when few token do trick."
- **Interactive question tool** — Tip: when Claude asks many questions, say `use the interactive question tool` for a better UX.

## Key Takeaways

1. **Sandcastle is the new Ralph runner** — Matt built and open-sourced it during the cohort. It replaces Docker Desktop with swappable sandboxes and a TypeScript API. Try it.
2. **Docker `sbx` > `docker sandbox`** — MicroVMs are lighter, have a TUI, and don't need Docker Desktop. One-line change in your AFK script.
3. **RTK saves 90%+ tokens** — If you're burning through Claude usage, this CLI proxy is a game-changer. 125M tokens saved by one user.
4. **Stay in the 80-100K smart zone** — Even with 1M context, the model's reasoning degrades past ~100K. Set `autoCompactWindow: 200000`.
5. **Automated guardrails > code review** — Trellis's approach: add lint rules and tests for every AI mistake. 50K unit tests + 9K integration tests = confidence to let agents work autonomously.
6. **Fresh context per iteration is key** — The real benefit of Ralph loops isn't parallelism, it's that each iteration starts clean.
7. **Security is a real concern for AFK** — Run agents in ephemeral environments. Assume breach. Lock down IAM. Rotate keys.
8. **Addy Osmani's skills are worth studying** — Very similar to Matt's setup, validates the approach from a Google engineering perspective.
